## 課題 3

### Storybook を使って開発することのメリット

- リアルタイムで要素の変更を確認できる
- 同じものを複数作られることを防ぐことができる
- ブラウザ上で動作して、導入済みのコンポーネントの確認が容易

### Storybook を使って開発することのデメリット

- コンポーネントが増えるたびに手動でアップデートが必要(管理コストがかかる)
- 最初の段階では開発に時間が掛かる
- 設計がまずいとコンポーネントが重くなる
